# Skill: Review thay đổi và tạo commit message

**Ngữ cảnh:** Sau khi hoàn thành một module hoặc một task.

**Nhiệm vụ:**
1. Tổng hợp tất cả các file đã tạo hoặc sửa trong phiên làm việc hiện tại.
2. Tóm tắt logic chính của các endpoint (hoặc chức năng) đã thêm.
3. Chỉ ra các điểm cần chú ý khi test (edge cases, điều kiện đặc biệt).
4. Đề xuất commit message theo format:  
   `feat(<tên-module>): <mô tả ngắn>`  
   hoặc  
   `fix(<tên-module>): <mô tả>`  
   hoặc  
   `chore: <mô tả>`
   `docs(<tên tài liệu>): <mô tả>.`
5. Liệt kệ các bước tôi cần làm trước khi commit (chạy lint, test thủ công, kiểm tra lại migration…).

**Đầu ra:** Bản tóm tắt thay đổi và commit message đề xuất.
