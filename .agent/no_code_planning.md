# No-Code Planning Rule

When the user asks for audit, planning, documentation, design review, backend mapping, or conflict fixing, **do not code**.

Only update documents, plans, contracts, and reports.

Before coding any feature, confirm that the user explicitly asked to implement code.

If you need to propose code-level changes during an audit or planning task, write them as plans, markdown diffs, or TODOs only. Never modify application code files (`.ts`, `.html`, `.css`) during a planning or audit request.
